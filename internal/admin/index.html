Close package

Use a period label that matches your books (e.g. 2026-04). Refresh the snapshot before month-end close so totals match what you file.

Good to know

  • Reconciliation: The Uses column should be reconciled against your source of truth (e.g. app analytics, backend tallies, or store reports). The export is a point-in-time snapshot — not revision-safe history unless you archive each file when you generate it.
  • Period label only drives the filename and in-file metadata. It does not change the API or anything in end users’ apps.
  • Remaining is computed as max − used when the code has a cap; otherwise it is empty / unlimited.
  • Join URL is the public link to /join/?code=… on this site — useful in internal docs and partner emails. Never publish secret bulk lists or admin-only data publicly.
  • Archiving: Store CSV/JSON exports under a dated folder structure (e.g. 2026/Q2/referral-pay/) and restrict access to internal staff only.

Before you export

  1. Confirm the period label matches the month or quarter you are closing.
  2. Click Refresh snapshot to load the latest codes from the server.
  3. Choose filters (inactive codes, minimum uses), then download CSV (Excel) or JSON (systems).
  4. Store the file in your internal archive; do not commit pay exports to the public site repo.
